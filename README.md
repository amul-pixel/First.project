# First.project
A simple game created with while,if, break and looping. 
Guessing game code :
I used sec_number ,guess_count as a variable. Where sec_number can be assumed as the number to be guessed. Where as guess_count can be stated to be as number of guesses. I set it to 0 as no guess has been made. Using a while loop an essential part required in almost every game.  I used guess_count as a condition which should be less than 3 . I didn't put an less than or equal to sign as it gives us up to 4 tries as guess_count has been set to 0.

Problems which arose during this program:
Python is extremely indent sensitive as such problem had occurred with break and if statement. I made a small mistake by not defining 3 as a guess limit as it would have been easier to change read and understand. It's not problematic for now but will cause a huge problem in large complex programs

Things I understood :
1. Better to define a number with a proper variable which will make program more understandable and easy to change as well as having it less complicated. 
2. Focus on indent as its very sensitive.
3. In while and if statements ,if is used to check condition is vaild or not only then proceed and while make sure that it repeats or loops till the condition is fulfilled.
4. While statement may have else part its not necessary that it always need an else

The biggest hurdle during this program was to use creativity , syntax and debugging, especially the indent problem which caused me a lot of trouble. A small program but a huge feat to a beginner.  I learned and took this lesson from the YouTube channel "Programming with Mosh".
